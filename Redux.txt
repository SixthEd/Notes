Redux : It is a state management library that allows you to have a global state
	means you can have state that is accessible from any component no matter where they are in the tree
	
	Concept 1: Store 
	It is the global state is going to be accessible across any component no matter where they are in the react application
	It is made of multiple slices , multiple slices of the state each responsible for a certain domain in the application
	
	Redux use the concept of immutablity
	we will never allow to mutate the state instead what reducer is going to do take the state they're giving me a copy of the state and then make those changes to that copy 
	which will also have all the other unchanged properties of the state and then will completely replace the state as a whole with the copy that has the changes applied

		    const todo = {id: nanoid(), text: action.payload.input};

		})
	    }
    

How Redux knows which reducer handles which action
	Short answer:

	Redux uses the type string inside the action.

	Every action looks like:

		{
		  type: "todo/add",
		  payload: { input: "Learn Redux" }
		}


		That type is the address of the reducer.

1️⃣ Slice creates action types automatically

	Your slice:

		const todoSlice = createSlice({
		  name: "todo",
		  reducers: {
		    add: (state, action) => {},
		    remove: (state, action) => {}
		  }
		});


		Redux Toolkit generates:

		"todo/add"
		"todo/remove"

2️⃣ Store combines reducers
	configureStore({
	  reducer: {
	    todo: todoReducer,
	    user: userReducer
	  }
	});


	Redux internally builds:

		rootReducer = combineReducers({
		  todo: todoReducer,
		  user: userReducer
		});

3️⃣ Dispatch flow
	dispatch(add({ input: "JS" }));


	Becomes:

		dispatch({
		  type: "todo/add",
		  payload: { input: "JS" }
		});


	Redux now does:

		rootReducer(state, action)


	Which calls:

		todoReducer(state.todo, action)
		userReducer(state.user, action)

4️⃣ Each reducer checks the action type

	Inside todoReducer:

		switch(action.type) {
		  case "todo/add":
		    // run add reducer
		    break;
		  case "todo/remove":
		    // run remove reducer
		    break;
		}


	Redux Toolkit hides this switch for you.

		Only the matching reducer executes.
		Others just return state unchanged.

5️⃣ Async thunk is the same
	addTodoAsync.pending   → "todo/addTodoAsync/pending"
	addTodoAsync.fulfilled → "todo/addTodoAsync/fulfilled"
	addTodoAsync.rejected  → "todo/addTodoAsync/rejected"


	Your extraReducers listens to those exact types.

	🧠 Mental Model
		dispatch(action)
			↓
		rootReducer
		   ├── todoReducer ← matches type → runs
		   └── userReducer ← no match → ignored


	Redux doesn’t “search” — it routes by string keys.

🎯 One-liner

	Redux doesn’t know reducers by name.
	It matches them by the action.type string.

	That’s the entire routing system.

		dispatch(addTodoAsync)
			↓
		Redux dispatches: todo/addTodoAsync/pending
			↓
		extraReducers.pending runs
			↓
		Async function runs (setTimeout, API)
			↓
		Promise resolves
			↓
		Redux dispatches: todo/addTodoAsync/fulfilled
			↓
		extraReducers.fulfilled runs
			↓
		state updated
			↓
		React re-renders
